# AZ-400: Designing and Implementing Microsoft DevOps Solutions

| Concepto                    | Qué es                                                                 | Para qué sirve                                                                 | Ejemplo práctico                                                                 |
|---------------------------|------------------------------------------------------------------------|--------------------------------------------------------------------------------|----------------------------------------------------------------------------------|
| DevOps Services           | Plataforma SaaS de Microsoft para desarrollo y operación               | Centralizar código, CI/CD, planificación y testing                             | Gestionar todo el ciclo de vida de una app en Azure                              |
| Pipelines (CI/CD)         | Sistema de automatización de builds y despliegues                      | Integración continua (CI) y despliegue continuo (CD)                            | Ejecutar Terraform al hacer push en Git                                          |
| Microsoft-hosted agents   | Máquinas gestionadas por Microsoft para ejecutar pipelines             | Ejecutar jobs sin mantener infraestructura propia                              | Runner temporal que ejecuta un `terraform apply`                                  |
| Self-hosted agents        | Máquinas propias (VM, server) que ejecutan pipelines                   | Control total, sin límites de minutos                                           | VM en Azure que ejecuta pipelines de forma continua                              |
| Artifacts                 | Repositorio de paquetes (NuGet, npm, Maven, etc.)                      | Almacenar y versionar dependencias o builds                                     | Guardar versiones de librerías internas                                          |
| Azure Boards              | Herramienta de gestión de trabajo (Agile, Kanban, Scrum)               | Planificación, seguimiento de tareas e incidencias                             | Backlog de features y bugs del equipo                                            |
| Test Plans                | Herramienta para testing manual y gestión de calidad                   | Definir, ejecutar y auditar tests                                               | QA ejecutando casos de prueba antes de release                                   |
